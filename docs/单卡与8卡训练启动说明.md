# 单卡与8卡训练启动说明

## 目的

当前项目已经同时支持：

- 单卡启动
- 8 卡 `torchrun` 启动

两者共用同一套训练代码，差别主要在启动脚本和分布式初始化方式。

## 启动脚本

### LoRA 训练

- 单卡：
  [run_geo_lora_train_1gpu.sh](/c:/DevelopProject/VScode/geomapgen/scripts/run_geo_lora_train_1gpu.sh)
- 8 卡：
  [run_geo_lora_train_8gpu.sh](/c:/DevelopProject/VScode/geomapgen/scripts/run_geo_lora_train_8gpu.sh)

### Full 训练

- 单卡：
  [run_geo_full_train_1gpu.sh](/c:/DevelopProject/VScode/geomapgen/scripts/run_geo_full_train_1gpu.sh)
- 8 卡：
  [run_geo_full_train_8gpu.sh](/c:/DevelopProject/VScode/geomapgen/scripts/run_geo_full_train_8gpu.sh)

## 单卡启动

### LoRA

```bash
bash scripts/run_geo_lora_train_1gpu.sh
```

如果想指定单卡编号：

```bash
GPU_ID=3 bash scripts/run_geo_lora_train_1gpu.sh
```

### Full

```bash
bash scripts/run_geo_full_train_1gpu.sh
```

如果想指定单卡编号：

```bash
GPU_ID=2 bash scripts/run_geo_full_train_1gpu.sh
```

## 8卡启动

### LoRA

```bash
bash scripts/run_geo_lora_train_8gpu.sh
```

### Full

```bash
bash scripts/run_geo_full_train_8gpu.sh
```

如果不是单机 8 卡，可以额外设置：

```bash
NNODES=2 NODE_RANK=0 MASTER_ADDR=10.0.0.1 MASTER_PORT=29601 NPROC_PER_NODE=8 bash scripts/run_geo_lora_train_8gpu.sh
```

## 当前8卡实现的工作方式

当前多卡训练不是把 patch 随机打散到不同卡，而是按 `sample_id` 分配到不同 rank。这样做是为了保住当前项目里很重要的两件事：

- 同一个 sample 内 patch 的顺序
- patch 间 state 连续性

相关实现位置：

- [train_geo_model.py](/c:/DevelopProject/VScode/geomapgen/unimapgen/train_geo_model.py)

## 8卡模式下的默认取舍

8 卡脚本默认更偏吞吐，不偏排查：

- 关闭大部分训练期中间资产导出
- 开启 cache
- 强制 `epoch_is_single_sample=false`

这是为了避免 8 卡训练被：

- patch 图导出
- 训练期预测导出
- 单 sample 串行训练

拖慢。

## 注意事项

### 1. 单卡和8卡不要混用同一个输出目录

建议为单卡和 8 卡分别设置不同的：

- `OUTPUT_DIR`
- `RUN_NAME`

### 2. 8卡模式下，不建议保留重型 debug 导出

如果你把这些重新打开：

- `SAVE_TRAIN_BATCH_GEOJSON=true`
- `SAVE_TRAIN_BATCH_PREDICTIONS=true`
- `ARTIFACT_EXPORT_ENABLED=true`

训练会明显变慢，8 卡也不容易吃满。

### 3. Full 训练显存压力远高于 LoRA

如果先验证链路，建议先跑：

- `run_geo_lora_train_1gpu.sh`
- `run_geo_lora_train_8gpu.sh`

确认数据、日志、预测都正常后，再上 full。

## 建议使用顺序

如果你是第一次切到 8 卡，建议按这个顺序：

1. 先用单卡 LoRA 跑通一轮，确认配置没问题
2. 再切 8 卡 LoRA，确认 DDP 正常
3. 最后再尝试 8 卡 full
