# 当前数据集 v1.0 适配与 8 卡训练说明

## 目标
这条链路把当前数据集：

- `/dataset/zsy/dataset-extracted`
- `patch_tif/0.tif`
- `patch_tif/0_edit_poly.tif`
- `label_check_crop/Lane.geojson`
- `label_check_crop/Intersection.geojson`

适配到 `unimapgen-v1.0` 的训练和 rollout 推理流程。

这里不再使用当前主项目的 `DINO + Qwen` 训练链，改为：

- `Qwen2.5-VL`
- `LLaMAFactory LoRA/SFT`
- `v1.0` 风格 line token 序列
- `prompt + previous state + image` rollout 推理

## 适配后的任务定义
### lane
`Lane.geojson` 直接转成 `v1.0` 的 line sequence 监督：

- `category`
- `start_type`
- `end_type`
- `points`

### intersection
`Intersection.geojson` 不按 polygon object 训练，而是转成：

- `intersection boundary lines`

也就是把路口边界作为线来训练和推理。这样可以完全复用 `v1.0` 的：

- cut endpoint
- state trace
- line token 序列

最终如果需要，再由边界线重建为 `Intersection.geojson`。

## 复用的当前项目思路
虽然训练架构切到了 `v1.0`，但数据裁切逻辑仍然参考当前项目：

- patch 尺寸、overlap、keep margin
- review-mask-aware 几何裁切
- patch 边界 cut 点监督
- sample 内扫描顺序
- 邻接 patch 的 state 传递

## 当前使用的主要参数
### manifest / tiling
- `tile_size_px=1024`
- `overlap_px=256`
- `keep_margin_px=128`
- `review_crop_pad_px=64`

### Stage A
- `lora_rank=16`
- `lora_alpha=32`
- `lora_dropout=0.05`
- `cutoff_len=8192`
- `learning_rate=1e-4`
- `num_train_epochs=3`
- `gradient_accumulation_steps=8`

### Stage B
- `learning_rate=5e-5`
- `num_train_epochs=2`
- 其余 LoRA 和 cutoff 沿用 Stage A 风格

### rollout
- `max_new_tokens=2048`
- `trace_points=8`
- 默认优先用 state-aware prompt
- 如果 state 为空，则可回退到 patch-only prompt

## 注意事项
### 1. 当前这条链不使用 DINO
虽然你之前也给了 DINO 权重路径，但 `v1.0` 这条路线用的是 `Qwen2.5-VL` 原生视觉塔，不会读取 DINO。

### 2. 8 卡训练依赖正确的 GPU PyTorch 环境
如果 `torch.cuda.is_available()` 为 `False`，8 卡训练不会启动成功。

### 3. 当前训练目标是 line-map
`intersection` 现在按 boundary lines 训练，而不是 polygon object 直接训练。

### 4. 真值验证主要在 rollout 阶段
训练脚本本身仍然是 LLaMAFactory SFT，GT 对齐验证主要看 rollout 输出的：

- `summary.json`
- `metrics/*.json`
- `visualizations/`
