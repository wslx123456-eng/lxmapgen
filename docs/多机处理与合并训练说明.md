# 多机处理与合并训练说明

## 依赖
只处理数据集：

```bash
pip install -r requirements.data.txt
```

训练与 rollout：

```bash
pip install -r requirements.train.txt
```

## 1. 多台机器分片处理数据集
每台机器都运行同一个脚本，只改分片参数：

```bash
export DATASET_ROOT=/dataset/zsy/dataset-extracted
export PROCESSED_ROOT=/your/output/root
export SHARD_INDEX=0
export NUM_SHARDS=4
bash scripts/run_geo_current_v1_process_dataset_shard.sh
```

第二台机器把 `SHARD_INDEX` 改成 `1`，后面依次类推。

处理完成后，每台机器会得到：

- `family_manifest.jsonl`
- `stage_a/dataset`
- `stage_b/dataset`

## 2. 合并多机处理结果
把所有机器产出的分片目录同步到同一台机器后执行：

```bash
export MERGED_ROOT=/your/merged/output
bash scripts/run_geo_current_v1_merge_processed_dataset.sh \
  /path/to/shard_00 \
  /path/to/shard_01 \
  /path/to/shard_02 \
  /path/to/shard_03
```

合并后会得到：

- `family_manifest.jsonl`
- `stage_a/dataset`
- `stage_b/dataset`
- `merge_summary.json`

## 3. 直接读取已处理数据训练
单卡：

```bash
export PROCESSED_ROOT=/your/merged/output
export TRAIN_OUTPUT_ROOT=/your/train/output
bash scripts/run_geo_current_v1_train_from_processed_1gpu.sh
```

8 卡：

```bash
export PROCESSED_ROOT=/your/merged/output
export TRAIN_OUTPUT_ROOT=/your/train/output
bash scripts/run_geo_current_v1_train_from_processed_8gpu.sh
```

## 可配置路径
这些脚本都支持通过环境变量改路径：

- `DATASET_ROOT`
- `PROCESSED_ROOT`
- `MERGED_ROOT`
- `TRAIN_OUTPUT_ROOT`
- `QWEN_ROOT`
- `MODEL_PATH`
- `PYTHON_BIN`
- `LLAMAFACTORY_BIN`

## 推荐流程
1. 每台机器各自跑 `run_geo_current_v1_process_dataset_shard.sh`
2. 汇总分片目录后跑 `run_geo_current_v1_merge_processed_dataset.sh`
3. 直接跑 `run_geo_current_v1_train_from_processed_1gpu.sh` 或 `run_geo_current_v1_train_from_processed_8gpu.sh`
