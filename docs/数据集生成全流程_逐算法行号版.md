# 数据集生成全流程（逐算法行号版）

这份文档只讲**当前代码里“生成数据集”这条主链**，不讲训练和推理。

目标是回答两件事：

1. 从原始 `train/val` 目录，到 `stage_a / stage_b / fixed16` 数据集，代码到底怎么走。
2. 每个关键算法现在在**哪个文件、哪一行**。

说明：

- 下文中的行号对应当前仓库代码状态。
- 只列**当前主链会走到的关键入口和关键算法**。
- 像调试辅助函数、旧版 raw-family-manifest 脚本，这里不展开。

---

## 1. 当前主链总览

当前最完整的一键入口是：

- [build_geo_current_manifest_and_datasets.py](/c:/DevelopProject/VScode/geomapgen/scripts/build_geo_current_manifest_and_datasets.py)

主函数：

- `main()` 第 **95** 行

它会按这个顺序执行：

1. 构建 `family_manifest.jsonl`
2. 从 manifest 导出：
   - `stage_a/dataset`
   - `stage_b/dataset`
3. 从 `stage_a/dataset` 再构建：
   - `fixed16_stage_a`
4. 从 `stage_b/dataset` 再构建：
   - `fixed16_stage_b`

可以把它理解成这条链：

```text
原始 train/val 根目录
-> build_manifest_for_dataset()
-> family_manifest.jsonl
-> export_families_to_stage_datasets()
-> stage_a/dataset + stage_b/dataset
-> build_fixed_grid_targetbox_dataset()
-> fixed16_stage_a + fixed16_stage_b
```

### 5.4 patch GT 几何裁切算法

核心函数：

这个函数是整条 GT 处理链里最重要的几何算法之一。

#### 4.4.2 Line 裁切

发生在：

- 第 **1380-1411** 行

调用：

- `clip_polyline_to_rect(...)`
  - 来自 [export_llamafactory_patch_only_from_raw_family_manifest.py](/c:/DevelopProject/VScode/geomapgen/scripts/export_llamafactory_patch_only_from_raw_family_manifest.py)

## 8. 当前主链里几个最常见的排查点

### 8.1 patch 为何被切出来

先看：

- `build_manifest_for_dataset(...)` 第 **1493** 行

再看：

- `generate_tile_windows(...)` 第 **939** 行
- `compute_mask_bbox(...)` 第 **195** 行
- `expand_bbox(...)` 第 **202** 行

### 8.2 某条 lane 为什么变成 cut

先看：

- `build_patch_segments_global(...)` 第 **1348** 行

重点看：

- `clip_polyline_to_rect(...)`
- `_line_piece_cut_flags_after_clip(...)` 第 **1161** 行
- `point_boundary_side(...)`

### 8.3 Stage B state 为什么这样生成

直接看：

- `extract_state_lines(...)` 第 **1783** 行
- `apply_state_mode(...)` 第 **1980** 行

### 8.4 fixed16 为什么不是 16 倍

直接看：

- `build_split(...)` 第 **546** 行
- `filter_pairs_to_empty_ratio(...)` 第 **452** 行

如果 `target-empty-ratio < 1.0`，empty box 会被下采样，所以不会固定翻 16 倍。
