# 当前数据集处理与验证启动说明

这份说明对应当前 `geo_current` 数据处理链，主入口是一键脚本：

- [build_geo_current_manifest_and_datasets.py](c:/DevelopProject/VScode/geomapgen/scripts/build_geo_current_manifest_and_datasets.py)

它会一次完成：

1. 构建 `family_manifest.jsonl`
2. 导出 `stage_a/dataset`
3. 导出 `stage_b/dataset`
4. 基于 `stage_a/dataset` 再构建 `fixed16_stage_a`
5. 基于 `stage_b/dataset` 再构建 `fixed16_stage_b`

## 一键处理

```powershell
python scripts/build_geo_current_manifest_and_datasets.py `
  --train-root "D:\your_raw_dataset\train" `
  --val-root "D:\your_raw_dataset\val" `
  --output-root "D:\output\processed_all" `
  --splits train val `
  --image-relpath "patch_tif/0.tif" `
  --mask-relpath "patch_tif/0_edit_poly.tif" `
  --lane-relpath "label_check_crop/Lane.geojson" `
  --intersection-relpath "label_check_crop/Intersection.geojson" `
  --tile-size-px 896 `
  --overlap-px 0 `
  --keep-margin-px 0 `
  --review-crop-pad-px 0 `
  --tile-min-mask-ratio 0.02 `
  --tile-min-mask-pixels 256 `
  --search-within-review-bbox `
  --fallback-to-all-if-empty `
  --resample-step-px 4.0 `
  --boundary-tol-px 2.5 `
  --trace-points 8 `
  --state-mixture-mode full `
  --state-no-state-ratio 0.30 `
  --state-weak-ratio 0.40 `
  --state-full-ratio 0.30 `
  --state-weak-trace-points 3 `
  --state-line-dropout 0.40 `
  --state-point-jitter-px 2.0 `
  --state-truncate-prob 0.30 `
  --empty-patch-drop-ratio 0.95 `
  --empty-patch-seed 42 `
  --fixed16-output-name "fixed16_stage_a" `
  --fixed16-stageb-output-name "fixed16_stage_b" `
  --fixed16-grid-size 4 `
  --fixed16-target-empty-ratio 0.10 `
  --fixed16-seed 42 `
  --fixed16-resample-step-px 4.0 `
  --fixed16-boundary-tol-px 2.5 `
  --fixed16-image-root-mode symlink `
  --use-system-prompt
```

输出目录：

- `D:\output\processed_all\family_manifest.jsonl`
- `D:\output\processed_all\stage_a\dataset`
- `D:\output\processed_all\stage_b\dataset`
- `D:\output\processed_all\fixed16_stage_a`

## 当前默认规则

### train

- `family_manifest` 会按你传入的：
  - `--search-within-review-bbox`
  - `--tile-min-mask-ratio`
  - `--tile-min-mask-pixels`
  - `--tile-max-per-sample`
  这些参数执行
- `stage_a / stage_b` 导出时，空 patch 会按：
  - `--empty-patch-drop-ratio`
  下采样
- `fixed16` 导出时，空 box 会按：
  - `--fixed16-target-empty-ratio`
  下采样

### val

一键脚本里已经做了专门覆盖，`val` 会走“完整输出”策略：

- 不按 `review bbox` 缩小搜索区域
- 不按 `tile_min_mask_ratio` / `tile_min_mask_pixels` 丢 patch
- 不按空 patch 丢 `stage_a / stage_b`
- `fixed16 val` 走全量 box：
  - 等价于 `allboxes`
  - 不会丢空 box

也就是说，`val` 的用途是：

- 保留它应该输出的全部 patch / 全部 box
- 后面推理后再用脚本拼回整图

## fixed16 推理结果拼回完整 `Lane.geojson`

脚本：

- [reconstruct_fixed16_predictions_to_lane_geojson.py](c:/DevelopProject/VScode/geomapgen/scripts/reconstruct_fixed16_predictions_to_lane_geojson.py)

这个脚本读取：

1. `fixed16` 的 `meta_val.jsonl`
2. 大模型对 `fixed16 val` 的预测结果
3. `family_manifest.jsonl`

然后做两步：

1. 先把 box 级预测回到 patch / 全图像素坐标
2. 再按 `cut -> cut` 端点把相邻 box / patch 的 lane 片段拼起来

### 拼 `fixed16_stage_a` 的 `val`

```powershell
python scripts/reconstruct_fixed16_predictions_to_lane_geojson.py `
  --fixed16-root "D:\output\processed_all\fixed16_stage_a" `
  --predictions-path "D:\output\predictions\fixed16_stage_a_val_predictions.jsonl" `
  --output-root "D:\output\fixed16_stage_a_val_stitched" `
  --split val
```

### 拼 `fixed16_stage_b` 的 `val`

```powershell
python scripts/reconstruct_fixed16_predictions_to_lane_geojson.py `
  --fixed16-root "D:\output\processed_all\fixed16_stage_b" `
  --predictions-path "D:\output\predictions\fixed16_stage_b_val_predictions.jsonl" `
  --output-root "D:\output\fixed16_stage_b_val_stitched" `
  --split val
```

输出目录里每个原始 sample 会有：

- `Lane.raw.geojson`
  - 只做 box -> 全图回投，不做跨 box 合并
- `Lane.geojson`
  - 做过 `cut` 端点拼接后的结果
- `reconstruct_fixed16_predictions.summary.json`

### 预测文件格式要求

脚本优先按 `id` 对齐预测结果；如果预测文件没有 `id`，但行数和 `meta_val.jsonl` 完全一致，也会按顺序对齐。

它兼容的常见字段有：

- `id`
- `predict`
- `prediction`
- `pred_text`
- `generated_text`
- `response`
- `output`
- `assistant`
- `pred_lines`

预测文本里只要能解析出：

```json
{"lines":[...]}
```

就可以。
